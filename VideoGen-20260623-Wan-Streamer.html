Wan-Streamer v0.1: End-to-end Real-time Interactive Foundation Models

📄 arXiv: 2606.25041 🏢 Alibaba / Wan-AI Team 📅 2026-06-23 提交 📂 CV / AI / Audio-Visual Interaction
Real-time Interaction End-to-End Full-Duplex Audio-Visual Streaming Block-Causal Attention Low Latency

📌 论文概要

Wan-Streamer 是一个原生流式、端到端的交互式基础模型，从零开始设计用于实时、低延迟、全双工音视频交互。在单一 Transformer 中同时建模语言、音频和视频作为输入和输出，通过 block-causal attention 协调增量式流式生成。

核心成就：模型侧响应延迟约 200 ms，总交互延迟约 550 ms（含 350 ms 网络延迟），支持亚秒级全双工音视频通信。流式单元最短 160 ms（25 fps）。

🏗️ 模型架构

统一 Transformer 架构

Wan-Streamer 在单一 Transformer 中统一处理所有模态：

维度设计
输入 token视觉 + 音频 + 文本（交错排列）
输出 token视觉 + 音频 + 文本（交错生成）
注意力机制Block-causal attention（增量式流式）
编码器因果编码器 (Causal Encoders)
解码器因果解码器 (Causal Decoders)
调度低延迟多模态 token 调度
关键设计：与级联式交互系统（分离的 VAD → ASR → LLM → TTS → 动画 → 视频生成）不同，Wan-Streamer 不依赖任何外部语言、语音、avatar 或视频生成模块。感知、推理、生成、响应时机、轮次管理和跨模态同步全部在统一模型中联合学习。

vs 级联系统

传统级联系统在每个模块边界都会累积延迟和错误。Wan-Streamer 的端到端设计消除了这些边界，减少了流水线延迟和错误累积。

🎯 关键特性

  • 全双工音视频交互：同时感知和生成，边听边回应
  • 亚秒级延迟：模型侧 ~200ms，总计 ~550ms（含网络）
  • 单一 Transformer：无需外部 ASR/TTS/动画/视频生成模块
  • 原生流式设计：因果编码器、因果解码器、block-causal attention 全栈可流式
  • 160ms 流式单元：25 fps 下最短 160ms 的流式单元，实现细粒度增量生成
  • 多角色演示：同一模型可生成不同人物、声音和场景的交互视频

⚙️ 技术细节

流式架构设计

  • 因果编码器：所有输入编码器都是因果的，确保流式处理时不看未来信息
  • 因果解码器：输出解码器也是因果的，支持增量式生成
  • Block-causal attention：输入和输出 token 按 block 组织，block 内可见前面的 token，跨 block 通过 causal mask 控制信息流
  • 多模态 token 调度：精心设计的调度策略，确保音视频同步

当前局限 (v0.1)

  • 分辨率: 192p（概念验证阶段）
  • 更高分辨率" readily scales" 但留待未来工作

💡 个人分析

核心贡献

  • 端到端统一：将感知、推理、生成、轮次管理全部统一在单一 Transformer 中，消除了级联系统的延迟和错误累积问题。这是交互式 AI 的架构突破
  • 全双工音视频：同时输出同步的音频和视频，且在亚秒级延迟内完成，这在开源模型中是首次
  • 原生流式设计：从编码器到解码器到注意力机制，全栈围绕可流式性重新设计，为实时交互 AI 提供了新的架构范式
  • vs GPT-4o Realtime：虽然 GPT-4o 延迟略低，但它是纯语音交互。Wan-Streamer 是唯一在相近延迟下同时输出视频的模型

技术亮点

  • Block-causal attention 的设计巧妙地统一了输入理解和输出生成
  • 160ms 流式单元 @ 25fps 实现了细粒度的增量响应
  • 多角色 demo 展示了同一模型的泛化能力（不同人物、声音、场景）

局限

  • v0.1 仅支持 192p 分辨率，是概念验证阶段
  • 论文较短（3.8MB），可能省略了大量训练细节
  • 实时交互场景的评估标准尚不完善
  • 作为最新发布（2026-06-23），社区反响和实际效果有待验证

方向意义

  • 代表 Wan 系列从视频生成向实时交互的扩展
  • 端到端 vs 级联的辩论在交互 AI 领域将持续，Wan-Streamer 为端到端方案提供了有力证据
  • 如果分辨率提升到 720P+，将直接影响虚拟人、视频通话、游戏 NPC 等场景

Wan-Streamer 架构：原生流式端到端多模态交互模型

核心创新

  • 原生流式设计：block-causal attention 增量流式，语言/音频/视频同时输入输出
  • 因果编码器/解码器：全栈因果设计，160ms 流式单元（25fps）
  • 无级联模块：不依赖 VAD/ASR/TTS/头像驱动/视频生成等外部模块
  • 亚秒延迟：模型端 ~200ms，总交互 ~550ms（含网络）

训练策略

  • 全栈从流式能力设计：因果编码器、因果解码器、block-causal attention、低延迟 token 调度
  • 联合训练语言/音频/视频的输入输出

损失函数

  • AR 损失（语言）+ 扩散损失（音频/视频）+ 轮替管理损失
  • 流式一致性约束
