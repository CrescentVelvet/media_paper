📌 论文概要

Wan2.2 是 Wan2.1 的重大升级版本，于 2025 年 7 月 28 日发布。虽然没有单独的 arXiv 论文（共享 arXiv:2503.20314），但通过 GitHub README 和技术博客发布了详细的更新信息。

核心升级：引入 MoE (Mixture-of-Experts) 架构到视频扩散模型中，通过跨时间步的去噪专家分离，在不增加计算成本的前提下大幅提升模型容量。

🏗️ 模型架构

MoE 架构创新

Wan2.2 的核心架构创新是将 Mixture-of-Experts (MoE) 引入视频扩散模型：

  • 专家分离：将去噪过程按时间步分配给不同的专家模型，每个专家专注于特定去噪阶段
  • 计算效率：整体模型容量增大，但推理时每个时间步只激活部分专家，保持相同计算成本
  • 架构继承：基于 Wan2.1 的 DiT 范式和 Wan-VAE，平滑升级

Wan2.2-VAE 升级

Wan2.2 引入了改进的 VAE，压缩比达到 16×16×4（空间 16×16，时间 4×），使 720P 视频生成更加高效。

🎯 关键特性

  • MoE 架构首次用于视频扩散：跨时间步专家分离，容量增大但成本不变
  • 电影级美学：精心筛选美学数据，带光照、构图、对比度、色调等详细标签，支持可定制的美学风格
  • 复杂运动生成：相比 Wan2.1，训练数据增加 65.6%（图像）和 83.2%（视频），运动、语义、美学多维泛化增强
  • 高效 720P TI2V：5B 模型使用 Wan2.2-VAE (16×16×4 压缩)，支持 720P@24fps，可在 RTX 4090 上运行，是最快的 720P@24fps 模型之一
  • 语音驱动视频 (S2V)：Wan2.2-S2V-14B 支持音频驱动的电影级视频生成（2025-08-26 发布）
  • 角色动画 (Animate)：Wan2.2-Animate-14B 统一角色动画与替换，支持整体运动和表情复制（2025-09-19 发布）

📊 基准表现

  • 在所有开源和闭源模型中达到 TOP 性能
  • Cosmos 3 论文中引用 Wan2.2-A14B 作为 Image2Video 基准（R-Bench: 81.3 分）
  • Tail-Aware HiFloat4 (arXiv: 2605.26628) 等量化论文以 Wan2.2 为目标模型，说明其已成为社区标准
  • 众多加速方法（Cache-dit、LightX2V、FastVideo 等）以 Wan2.2 为优化目标
行业影响：Wan2.2 已成为视频生成领域的事实标准之一。大量社区工作围绕 Wan2.2 展开：量化加速（HiFloat4）、缓存优化（Cache-dit）、稀疏注意力（LVSA）、角色动画（HuMo）等。

💡 个人分析

核心贡献

  • MoE 引入视频扩散：首创性地将 MoE 架构引入视频扩散模型，通过时间步专家分离实现"容量增大但成本不变"，为视频生成模型的规模化提供了新路径
  • 美学可控生成：带详细美学标签的数据策略使模型支持可定制的电影风格，这是区别于其他模型的重要差异化能力
  • 多模态扩展：从 T2V/I2V 扩展到 S2V（语音驱动）和 Animate（角色动画），展示了架构的通用性
  • 高效 720P：TI2V-5B 模型在消费级 GPU 上实现 720P@24fps，平衡了质量与效率

值得注意

  • 没有独立论文发布，技术细节通过 GitHub 和博客公开，这在快速迭代的 AI 领域是常见做法
  • MoE 架构在视频扩散中的有效性已被社区验证（多个量化/加速论文基于 Wan2.2 MoE）
  • Wan2.2-VAE 被 NVIDIA Cosmos 3 直接采用作为视觉编码器，说明 VAE 设计已获业界顶级认可

局限

  • A14B MoE 模型总参数量较大，虽然激活参数可控但仍需 80GB+ VRAM 进行高效推理
  • S2V 和 Animate 功能虽然有 demo，但应用场景仍在探索中
