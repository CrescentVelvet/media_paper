📌 论文概要

Wan2.2 是 Wan2.1 的重大升级版本，于 2025 年 7 月 28 日发布。虽然没有单独的 arXiv 论文（共享 arXiv:2503.20314），但通过 GitHub README 和技术博客发布了详细的更新信息。

核心升级：引入 MoE (Mixture-of-Experts) 架构到视频扩散模型中，通过跨时间步的去噪专家分离，在不增加计算成本的前提下大幅提升模型容量。

🏗️ 模型架构

MoE 架构创新

Wan2.2 的核心架构创新是将 Mixture-of-Experts (MoE) 引入视频扩散模型：

  • 专家分离：将去噪过程按时间步分配给不同的专家模型，每个专家专注于特定去噪阶段
  • 计算效率：整体模型容量增大，但推理时每个时间步只激活部分专家，保持相同计算成本
  • 架构继承：基于 Wan2.1 的 DiT 范式和 Wan-VAE，平滑升级

Wan2.2-VAE 升级

Wan2.2 引入了改进的 VAE，压缩比达到 16×16×4（空间 16×16，时间 4×），使 720P 视频生成更加高效。

🖼️ 算法框图

注: 架构图请参考 Wan2.2 GitHub README 和技术博客

Wan2.2 的核心架构与 Wan2.1 的主要差异在于 MoE 引入:

  • MoE 路由: 不同去噪时间步分配给不同专家, 每个专家专注于特定去噪阶段(早期/中期/晚期)
  • Wan2.2-VAE: 升级的 VAE, 压缩比 16×16×4, 比 Wan2.1 更高效
  • 共享主干: MoE 层替换部分 FFN 层, 注意力层保持共享, 减少参数冗余
  • 推理路由: 给定时间步 t, 路由器选择对应的专家, 其他专家不被激活

首创性地将 Mixture-of-Experts 架构引入视频扩散模型, 通过跨时间步的去噪专家分离, 在不增加推理计算成本的前提下大幅提升模型容量, 为视频生成模型的规模化提供了新路径。

🏋️ 训练策略

预训练阶段

  • 基础继承: 从 Wan2.1 预训练权重初始化, 平滑升级到 MoE 架构
  • MoE 训练: 专家路由器从零训练, 专家权重从 Wan2.1 FFN 权重复制后分化
  • 渐进式容量增长: 先训练少量专家(2-4), 逐步增加到更多专家
  • 负载均衡: 使用 auxiliary loss 确保各专家负载均衡, 避免专家退化
  • 美学数据注入: 带详细美学标签的电影数据在后训练阶段注入

后训练阶段

  • SFT: 任务特定微调(T2V, I2V, S2V, Animate)
  • 美学偏好对齐: 使用带美学标签的数据进行 DPO/偏好对齐
  • 音频对齐 (S2V): 音频特征与视频帧的时序对齐训练

🔬 损失函数

Flow Matching 损失 (继承 Wan2.1)

  • L_FM = E_{t,x0,ε} ||v_θ^{expert(t)}(x_t, t, c) - u_t||²
  • 区别: v_θ^{expert(t)} 根据时间步 t 路由到不同专家, 每个专家专精特定去噪阶段

MoE 辅助损失

  • 负载均衡损失: L_balance = N · Σ f_i · P_i, 鼓励各专家均匀使用
  • 其中 f_i 是专家 i 的输入比例, P_i 是路由器对专家 i 的平均概率
  • 总损失: L = L_FM + α · L_balance

VAE 损失 (Wan2.2-VAE)

  • 重建 L1 + LPIPS + 对抗损失 + KL 散度
  • 相比 Wan-VAE, 压缩率提升到 16×16×4, 减少了 token 数量
MoE 在视频扩散中的核心优势:视频生成的去噪过程天然具有阶段差异(早期去噪结构, 晚期去噪细节), MoE 让不同专家专精不同阶段, 提升整体质量而不增加推理成本。
