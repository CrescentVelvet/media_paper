📌 论文概要

Cosmos 3 是 NVIDIA 提出的全模态世界模型家族,能在统一的 Mixture-of-Transformers (MoT) 架构中联合处理和生成语言、图像、视频、音频和动作序列。通过灵活的输入输出配置,将视觉语言模型、视频生成器、世界模拟器和世界动作模型统一到单一框架中。

核心成就:后训练模型在 Artificial Analysis 评测中被评为最佳开源 Text-to-Image 和 Image-to-Video 模型,在 RoboArena 中被评为最佳策略模型。在多项理解和生成任务中达到 SOTA。

🏗️ 模型架构(深入)

Cosmos 3 采用统一的 Mixture-of-Transformers (MoT) 架构。输入序列分为两个子序列:

子序列包含注意力机制路由
AR 子序列语言 token + ViT 编码的视觉 token因果自注意力(只能看到前面的 token)Reasoner 塔
DM 子序列VAE 编码的视觉/音频/动作 token(加噪)全双向注意力(可看到 AR+DM 所有 token)Generator 塔
关键设计:AR token 永远不会被 DM token 更新(保持因果完整性),但 DM token 可以自由注意到 AR 的文本提示。两个塔共享同一个注意力层,但有独立的 LayerNorm、MLP 和投影矩阵。两者均从预训练 VLM 权重初始化。

编码器

模态编码器压缩率训练状态
视觉(理解)ViT (16×16 patch + 2×2 merge)空间 32×32 压缩与主干联合训练
视觉(生成)Wan2.2 视频 VAE时间 4× / 空间 32×32冻结
音频Audio VAE (Lee et al.)48kHz → 25 token/s冻结
动作Domain-aware 线性投影按 embodiment 维度从零初始化,联合训练

统一动作表示

所有 embodiment 的动作都映射为 ego pose (9D) + effector pose (9D) + grasp state 的组合:

  • 相机运动 (9D):仅 ego pose
  • 自动驾驶 (9D):仅 ego pose
  • 自我中心运动 (57D):ego(9D) + 双腕 effector(9D×2) + 指尖 grasp(15D×2)
  • 单臂机器人 (10D):ego(9D) + gripper open/close(1D)
  • 双臂机器人 (20D):单臂 × 2
  • 人形机器人 (29D):ego(9D) + 双臂(9D+1D)×2

ego/effector pose 使用 SE(3) 相对变换的 6D 旋转表示(Zhou et al. 2019),grasp state 直接编码当前操作状态。不同 embodiment 使用独立的输入/输出投影矩阵,但共享 MoT 主干。

3D 多维旋转位置嵌入 (mRoPE)

扩展 Qwen3-VL 的 3D MRoPE,增加绝对时间索引来对齐不同模态的时间轴。原始 MRoPE 的 temporal 分量只记录 token 离散索引,无法处理视频/音频/动作以不同帧率同时生成的情况。Cosmos 3 将所有模态对齐到同一物理时间轴。

Token 排列规则

  • AR token 在前,DM token 在后
  • DM 内部:clean 条件 token 在前,noisy 生成 token 在后
  • 模态顺序:视觉 → 音频 → 动作
  • 特殊 token:<EOS> 结束 AR 序列,<BOG> (Begin-of-Generation) 开始 DM 序列

🎯 核心能力

1. 世界理解 (World Understanding)

  • 视频/图像描述、时序事件检测、下一步动作预测
  • 空间定位 (2D grounding)、物理合理性判断、因果推理

2. 世界生成 (World Generation)

  • 文本→图像/视频,图像→视频,视频→视频
  • 同步音频生成,动作条件视频 rollout

3. 动作建模 (Action Modeling)

  • 前向动力学:给定观察+动作 → 预测未来视频
  • 策略模型:给定观察+指令 → 输出动作轨迹
  • 逆动力学:给定视频 → 推断动作轨迹

4. 推理+生成联合

先对场景推理生成动作轨迹,再用该结构化意图驱动物理交互视频生成(Plan first, then bring it to life)。

💡 深度分析

核心贡献与意义

  • 架构统一：用单一 MoT 架构统一了理解(AR)和生成(Diffusion)，避免了多模型拼装的复杂性。双塔结构 + 共享注意力让两种范式既能独立训练又能互相增强
  • Physical AI 范式：将世界模型从“看”和“说”扩展到“做”，打通了感知→推理→动作→模拟的完整闭环
  • 动作作为一等公民：Action token 被设计为连接视觉帧的因果变量 (a_t 连接 v_{t-1} 和 v_t)，统一了 forward dynamics / inverse dynamics / policy 三种模式
  • Embodment 泛化：通过 domain-aware 投影层，同一主干可以处理从 9D 相机到 29D 人形机器人的各种动作空间
  • 开源生态：代码、权重、5 类合成数据集、评测基准全开放（OpenMDW-1.1 许可），对学术界和初创公司极为友好

技术亮点

  • 绝对时间轴 mRoPE：解决了不同模态不同帧率的时间对齐问题
  • Token 排列规则：AR在前 DM在后，条件 token 在前 noisy token 在后，统一支持所有生成模式
  • 双塔均从预训练 VLM 初始化：继承语言和视觉推理能力的同时学习生成
  • 合成数据集 (SDG)：PhyxSim / RobotSim / DriveSim / SynHuman / Warehouse 五类，消融实验证明有效
  • 推理+生成联合 pipeline：先推理生成动作轨迹，再驱动物理交互视频生成

值得关注

  • Reasoner 对 Generator 的增益（Appendix E.1 消融实验）
  • 多动作模式之间的协同效应（Appendix E.4）
  • 视频-动作一致性评估（Appendix E.5）
  • Cosmos3-Edge 轻量模型（Appendix D）的训练方法
  • Cosmos-HumanEval (HUE) 评测基准的设计（Appendix F）

局限

  • 部署门槛高：需要 NVIDIA GPU + Linux，最小 16B 模型也需要可观显存
  • Generator 部分功能（video-to-video、action）在 vLLM-Omni 中仍在开发中
  • General 类别推理不及 Gemini 3.1 Pro（73.7 vs 77.5），可能在纯文本/通用知识方面有差距
  • 音频生成质量略低于 Veo-3.1（7.31 vs 7.45）
  • 论文 139 页，200+ 作者，技术细节非常密集，需要分章节深入阅读

Figure 2: 整体架构概览

架构流水线:
  • Step 1 — 多模态编码: 文本→LLM tokenizer (Qwen3-VL 词表), 图像/视频(理解)→ViT 16×16 patch + 2×2 merge → 32×32 空间压缩, 图像/视频(生成)→Wan2.2 视频 VAE (时间4× / 空间32×32), 音频→Audio VAE (48kHz → 25 token/s), 动作→Domain-aware 线性投影 (按 embodiment 维度)
  • Step 2 — Token 排列: AR token (语言 + 视觉理解) 在前 → <EOS> 结束 AR 序列 → <BOG> 开始 DM 序列 → DM clean 条件 token → DM noisy 生成 token → 模态顺序: 视觉 → 音频 → 动作
  • Step 3 — 双塔 MoT 处理: Reasoner 塔 (因果自注意力, 只看前方) + Generator 塔 (全双向注意力, 可见 AR + DM), 共享注意力层但独立 LayerNorm/MLP/投影矩阵
  • Step 4 — 输出解码: AR→文本 token 自回归生成; DM→Flow Matching 速度场预测 → 积分去噪 → VAE 解码 → 视觉/音频/动作输出

Figure 3: Mixture-of-Transformers (MoT) 双塔结构

双塔信息流:
  • Reasoner 塔 (AR): 处理语言 token + ViT 编码的视觉 token, 使用因果掩码 (causal mask), 只能看到前面的 token。从预训练 VLM (Qwen3-VL) 权重初始化。输出: 文本回复、2D grounding 坐标、动作轨迹预测
  • Generator 塔 (DM): 处理 VAE 编码的视觉/音频/动作 token (加噪), 使用全双向注意力 (可见 AR + DM 所有 token)。同样从 VLM 权重初始化。输出: Flow Matching 速度场 → 去噪生成
  • 关键约束: AR token 永远不会被 DM token 更新 (因果掩码保证梯度隔离), 但 DM token 可以自由注意到 AR 的文本提示, 实现条件生成
  • 共享 vs 独立: Self-Attention 层共享 (同一矩阵), 但 LayerNorm、MLP、Input/Output 投影矩阵独立, 两塔有不同的学习动态

Figure 4: 统一动作表示与多 Embodiment 映射

动作统一编码: 所有 embodiment 的动作都映射为 ego pose (9D) + effector pose (9D) + grasp state 的组合, 使用 SE(3) 6D 旋转表示 (Zhou et al. 2019)。不同 embodiment 使用独立的输入/输出投影矩阵, 但共享 MoT 主干。
  • 相机运动 (9D): 仅 ego pose [R(6D) + t(3D)]
  • 自动驾驶 (9D): 仅 ego pose [R(6D) + t(3D)]
  • 自我中心运动 (57D): ego(9D) + 双腕 effector(9D×2) + 指尖 grasp(15D×2)
  • 单臂机器人 (10D): ego(9D) + gripper open/close(1D)
  • 双臂机器人 (20D): 单臂 × 2
  • 人形机器人 (29D): ego(9D) + 双臂(9D+1D)×2

3D 多维旋转位置嵌入 (mRoPE) 详解

Cosmos 3 扩展 Qwen3-VL 的 3D MRoPE, 增加绝对时间索引来对齐不同模态的时间轴:

  • 原始 MRoPE: temporal 分量只记录 token 离散索引 (第 1 帧、第 2 帧...), 无法处理视频 24FPS + 音频 25 token/s + 动作 10Hz 的不同帧率
  • Cosmos 3 mRoPE: 所有模态对齐到同一物理时间轴 (秒), 例如 t=2.5s 时, 视频对应第 60 帧, 音频对应第 62 个 token, 动作对应第 25 步
  • 位置编码维度分配: temporal (时间) + spatial_h (空间行) + spatial_w (空间列), 每个维度独立旋转

💡 核心创新点

创新 1: Mixture-of-Transformers (MoT) 统一架构
首次在单一 Transformer 框架中通过双塔结构(Reasoner + Generator)统一自回归理解和扩散生成, 两者共享注意力层但保持独立的 MLP 和 LayerNorm, 实现了理解和生成的端到端联合训练。
创新 2: 绝对时间轴 mRoPE
扩展 Qwen3-VL 的 3D MRoPE, 增加绝对物理时间索引, 解决了视频/音频/动作以不同帧率同时生成的时间对齐问题, 这是多模态世界模型的关键技术突破。
创新 3: 统一动作表示
将从 9D 相机运动到 29D 人形机器人的各种动作空间统一映射为 ego pose + effector pose + grasp state 组合, 使用 SE(3) 6D 旋转表示, 共享主干但独立投影层, 实现跨 embodiment 泛化。
创新 4: 推理+生成联合 Pipeline
"Plan first, then bring it to life" — 先用 Reasoner 推理场景并生成动作轨迹, 再用 Generator 据此驱动物理交互视频生成, 打通了感知→推理→动作→模拟的完整闭环。
创新 5: 五类合成数据集 (SDG)
PhyxSim(物理交互), RobotSim(机器人操作), DriveSim(自动驾驶), SynHuman(人类活动), Warehouse(仓储物流), 通过消融实验证明对模型能力提升有显著贡献。

🏋️ 训练策略（深入）

阶段 1: 预训练 — 大规模多模态学习

  • 主干初始化: Reasoner 和 Generator 塔均从预训练 VLM (Qwen3-VL 系列) 权重初始化, 继承语言和视觉推理能力。这是关键设计 — 让生成塔也能"理解"文本语义
  • 渐进式训练: 从低分辨率(256p)到高分辨率(720p), 从短视频(5帧)到长视频(300帧), 逐步增加上下文窗口
  • 多模态逐步加入: 文本+图像 → 视频 → 音频 → 动作, 每个阶段引入新模态时使用较小的学习率
  • Flow Matching: DM 子序列使用 Rectified Flow Matching 进行训练, 从噪声到数据的连续时间生成, 训练时 t ~ U[0,1]
  • 数据配比: 图像和视频按一定比例混合训练, 确保图像生成质量不退化; 动作数据与视觉数据按 embodiment 类型分组训练

阶段 2: 后训练 — 任务对齐与能力强化

  • SFT (监督微调): 针对特定任务(T2I, I2V, Policy 等)使用高质量标注数据进行微调, 包括美学评分数据、机器人示教数据、驾驶场景数据
  • RL (强化学习): 使用 RLHF/RLAIF 对 Reasoner 进行对齐, 提升推理质量和安全性; 对 Generator 使用人类偏好反馈优化生成质量

🔬 损失函数（深入）

AR 子序列 (Reasoner) — 自回归语言建模

  • 标准语言建模损失: L_AR = -Σ log P(x_t | x_{, 对语言和视觉理解 token 使用标准交叉熵
  • 视觉 token: ViT 编码的视觉特征也参与 AR 损失, 但使用蒸馏目标(与教师模型对齐)
  • 动作策略损失: 当 AR 子序列包含动作 token 时, 使用行为克隆损失 L_policy = -Σ log π(a_t | s_{, 其中 s 是视觉+文本观察

DM 子序列 (Generator) — Flow Matching

  • Rectified Flow Matching 损失: L_FM = E_{t,x0,ε} ||v_θ(x_t, t, c) - (x1 - x0)||²
  • v_θ 是模型预测的速度场 (velocity field), 参数化为 MoT 的 Generator 塔
  • x_t = (1-t)·x0 + t·x1, t ∈ [0,1], x0 ~ N(0,I) 是高斯噪声, x1 是真实数据 (VAE 编码后的潜在表示)
  • c 是条件 (AR token 的输出), 通过共享注意力层注入 DM token — 无需额外交叉注意力编码器
  • Rectified Flow 优势: 相比标准 DDPM, 推理时可用更少步数 (10-30 步) 生成高质量结果, 且训练更稳定

CFG (Classifier-Free Guidance)

  • 训练时: 随机丢弃条件 (10% 概率将 AR 条件置为零), 学习无条件分布
  • 推理时: 使用正负 CFG 分支并行加速: v_guided = v_uncond + s·(v_cond - v_uncond), s 为引导尺度
  • vLLM-Omni 实现: 支持 --cfg-parallel-size 2 在 2 个 GPU 上并行运行正负 CFG 分支

联合训练

  • 总损失: L_total = λ_AR · L_AR + λ_DM · L_FM, 两个损失同时反向传播
  • 梯度隔离: AR token 永远不被 DM token 更新 (因果掩码保证), 但 DM token 可以自由注意到 AR 的文本提示
  • 动作双向约束: 动作 token 同时参与 AR (策略: 观察→动作) 和 DM (动力学: 观察+动作→未来视觉) 损失, 实现双向约束
  • 多任务权重调度: λ_AR 和 λ_DM 在不同训练阶段动态调整, 预训练阶段以 AR 为主, 后训练阶段增加 DM 权重
