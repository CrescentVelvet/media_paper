📌 论文概要

Cosmos 3 是 NVIDIA 提出的全模态世界模型家族,能在统一的 Mixture-of-Transformers (MoT) 架构中联合处理和生成语言、图像、视频、音频和动作序列。通过灵活的输入输出配置,将视觉语言模型、视频生成器、世界模拟器和世界动作模型统一到单一框架中。

核心成就:后训练模型在 Artificial Analysis 评测中被评为最佳开源 Text-to-Image 和 Image-to-Video 模型,在 RoboArena 中被评为最佳策略模型。在多项理解和生成任务中达到 SOTA。

🏗️ 模型架构(深入)

Cosmos 3 采用统一的 Mixture-of-Transformers (MoT) 架构。输入序列分为两个子序列:

子序列包含注意力机制路由
AR 子序列语言 token + ViT 编码的视觉 token因果自注意力(只能看到前面的 token)Reasoner 塔
DM 子序列VAE 编码的视觉/音频/动作 token(加噪)全双向注意力(可看到 AR+DM 所有 token)Generator 塔
关键设计:AR token 永远不会被 DM token 更新(保持因果完整性),但 DM token 可以自由注意到 AR 的文本提示。两个塔共享同一个注意力层,但有独立的 LayerNorm、MLP 和投影矩阵。两者均从预训练 VLM 权重初始化。

编码器

模态编码器压缩率训练状态
视觉(理解)ViT (16×16 patch + 2×2 merge)空间 32×32 压缩与主干联合训练
视觉(生成)Wan2.2 视频 VAE时间 4× / 空间 32×32冻结
音频Audio VAE (Lee et al.)48kHz → 25 token/s冻结
动作Domain-aware 线性投影按 embodiment 维度从零初始化,联合训练

统一动作表示

所有 embodiment 的动作都映射为 ego pose (9D) + effector pose (9D) + grasp state 的组合:

  • 相机运动 (9D):仅 ego pose
  • 自动驾驶 (9D):仅 ego pose
  • 自我中心运动 (57D):ego(9D) + 双腕 effector(9D×2) + 指尖 grasp(15D×2)
  • 单臂机器人 (10D):ego(9D) + gripper open/close(1D)
  • 双臂机器人 (20D):单臂 × 2
  • 人形机器人 (29D):ego(9D) + 双臂(9D+1D)×2

ego/effector pose 使用 SE(3) 相对变换的 6D 旋转表示(Zhou et al. 2019),grasp state 直接编码当前操作状态。不同 embodiment 使用独立的输入/输出投影矩阵,但共享 MoT 主干。

3D 多维旋转位置嵌入 (mRoPE)

扩展 Qwen3-VL 的 3D MRoPE,增加绝对时间索引来对齐不同模态的时间轴。原始 MRoPE 的 temporal 分量只记录 token 离散索引,无法处理视频/音频/动作以不同帧率同时生成的情况。Cosmos 3 将所有模态对齐到同一物理时间轴。

Token 排列规则

  • AR token 在前,DM token 在后
  • DM 内部:clean 条件 token 在前,noisy 生成 token 在后
  • 模态顺序:视觉 → 音频 → 动作
  • 特殊 token:<EOS> 结束 AR 序列,<BOG> (Begin-of-Generation) 开始 DM 序列

🎯 核心能力

1. 世界理解 (World Understanding)

  • 视频/图像描述、时序事件检测、下一步动作预测
  • 空间定位 (2D grounding)、物理合理性判断、因果推理

2. 世界生成 (World Generation)

  • 文本→图像/视频,图像→视频,视频→视频
  • 同步音频生成,动作条件视频 rollout

3. 动作建模 (Action Modeling)

  • 前向动力学:给定观察+动作 → 预测未来视频
  • 策略模型:给定观察+指令 → 输出动作轨迹
  • 逆动力学:给定视频 → 推断动作轨迹

4. 推理+生成联合

先对场景推理生成动作轨迹,再用该结构化意图驱动物理交互视频生成(Plan first, then bring it to life)。

💡 深度分析

核心贡献与意义

  • 架构统一：用单一 MoT 架构统一了理解(AR)和生成(Diffusion)，避免了多模型拼装的复杂性。双塔结构 + 共享注意力让两种范式既能独立训练又能互相增强
  • Physical AI 范式：将世界模型从“看”和“说”扩展到“做”，打通了感知→推理→动作→模拟的完整闭环
  • 动作作为一等公民：Action token 被设计为连接视觉帧的因果变量 (a_t 连接 v_{t-1} 和 v_t)，统一了 forward dynamics / inverse dynamics / policy 三种模式
  • Embodment 泛化：通过 domain-aware 投影层，同一主干可以处理从 9D 相机到 29D 人形机器人的各种动作空间
  • 开源生态：代码、权重、5 类合成数据集、评测基准全开放（OpenMDW-1.1 许可），对学术界和初创公司极为友好

技术亮点

  • 绝对时间轴 mRoPE：解决了不同模态不同帧率的时间对齐问题
  • Token 排列规则：AR在前 DM在后，条件 token 在前 noisy token 在后，统一支持所有生成模式
  • 双塔均从预训练 VLM 初始化：继承语言和视觉推理能力的同时学习生成
  • 合成数据集 (SDG)：PhyxSim / RobotSim / DriveSim / SynHuman / Warehouse 五类，消融实验证明有效
  • 推理+生成联合 pipeline：先推理生成动作轨迹，再驱动物理交互视频生成

值得关注

  • Reasoner 对 Generator 的增益（Appendix E.1 消融实验）
  • 多动作模式之间的协同效应（Appendix E.4）
  • 视频-动作一致性评估（Appendix E.5）
  • Cosmos3-Edge 轻量模型（Appendix D）的训练方法
  • Cosmos-HumanEval (HUE) 评测基准的设计（Appendix F）

局限

  • 部署门槛高：需要 NVIDIA GPU + Linux，最小 16B 模型也需要可观显存
  • Generator 部分功能（video-to-video、action）在 vLLM-Omni 中仍在开发中
  • General 类别推理不及 Gemini 3.1 Pro（73.7 vs 77.5），可能在纯文本/通用知识方面有差距
  • 音频生成质量略低于 Veo-3.1（7.31 vs 7.45）
