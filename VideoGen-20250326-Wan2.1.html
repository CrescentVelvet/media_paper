📌 论文概要

Wan2.1 是阿里巴巴 Wan-AI 团队推出的开源视频基础模型套件，基于主流的 Diffusion Transformer (DiT) 范式构建。论文系统性地介绍了从 VAE 设计、可扩展预训练策略、大规模数据清洗到自动评估指标的一系列创新，在多个基准测试中超越了当时的开源模型和商业解决方案。

四大核心特征：① 领先性能（14B 模型在数十亿图片/视频上训练，展现 scaling law）；② 全面性（1.3B 和 14B 两个规格，覆盖 T2V、I2V、视频编辑、个人视频生成等 8 种任务）；③ 消费级效率（1.3B 模型仅需 8.19 GB VRAM）；④ 完全开源（代码 + 模型权重全部开放）。

🖼️ 算法框图与架构详解

Figure 1: 整体架构概览

架构流水线:
  • Step 1 — 输入编码: 文本→umT5-XXL 编码器 (多语言, 输出 4096 维文本特征); 视频/图像→Wan-VAE 编码器 (3D 因果卷积, 时间 4× / 空间 16×16 压缩)
  • Step 2 — 条件注入: 文本特征通过交叉注意力 (cross-attention) 注入 DiT 主干, 每个 Transformer 均包含 self-attention + cross-attention + MLP
  • Step 3 — DiT 主干: Full-attention Transformer + 3D RoPE (时间 + 空间行 + 空间列), 处理时空联合 token 序列, 无需分帧处理
  • Step 4 — Flow Matching: 从高斯噪声到视频数据的连续时间生成, 使用 Rectified Flow (线性插值路径)
  • Step 5 — 输出: DiT 预测速度场 v_θ → ODE 积分 (10-30 步) → Wan-VAE 解码 → 像素视频

Figure 2: Wan-VAE 结构

Wan-VAE (3D 因果 VAE) 详解:
  • 编码器: 输入 RGB 视频 → 3D 因果卷积 (时间维只能看到过去帧) → 多层 ResBlock 下采样 → 潜在表示。时间 4× 下采样, 空间 16×16 下采样
  • 解码器: 潜在表示 → 多层 ResBlock 上采样 → 3D 转置卷积 → 重建 RGB 视频
  • 因果性设计: 时间维使用因果掩码, 确保帧 t 的编码只依赖帧 ≤ t, 支持自回归式视频生成
  • 性能: 1080P 视频编解码 PSNR > 36dB, 压缩率 4×16×16 = 1024× (时间×空间×空间)
  • 影响: Wan-VAE 被 Wan2.2, Cosmos 3 (NVIDIA) 直接采用, 成为视频生成的事实标准 VAE

Figure 3: 3D RoPE 位置编码

3D RoPE ( Rotary Position Embedding) 详解:
  • 将旋转位置嵌入扩展到 3D: 时间维 (T) + 空间行维 (H) + 空间列维 (W)
  • 每个 token 的位置由 (t, h, w) 三元组确定, 通过不同频率的旋转编码
  • 优势: 支持任意分辨率和帧数 (无需固定位置表), 自然处理时空关系, 比加法式位置嵌入更好
  • 在 full-attention 中, 3D RoPE 让模型能感知 token 之间的时空相对距离

🏋️ 训练策略（深入）

阶段 1: 预训练 — 大规模视频学习

  • 渐进式训练: 从低分辨率(256p, 5帧)到高分辨率(720p/1080p, 300帧), 分 4 个阶段逐步提升。每个阶段训练到收敛后再提升分辨率
  • Flow Matching: 使用 Rectified Flow 作为训练目标, 从噪声到数据的连续时间生成。训练时 t ~ U[0,1], 采样路径为线性插值
  • 3D RoPE: 旋转位置嵌入扩展到 3D(时间+空间行+空间列), 处理时空联合注意力。比加法式位置嵌入更好地捕获时空相对关系
  • umT5-XXL 文本编码器: 多语言支持的文本理解, 提供丰富的语义条件。umT5-XXL 参数冻结, 不参与训练
  • 数据配比: 图像和视频按一定比例混合训练, 确保图像生成质量不退化。1.3B 模型图像:视频 ≈ 1:1, 14B 模型视频比例更高
  • 批量大小与学习率: 14B 模型使用 256 批量大小, 学习率 1e-4 → 1e-5 余弦衰减; 1.3B 模型使用 512 批量大小

阶段 2: 后训练 — 质量提升与任务适配

  • 任务微调: 针对特定任务(I2V, 视频编辑, FLF2V 等)使用高质量数据进行微调。I2V 训练时将首帧作为条件注入
  • 美学对齐: 通过美学评分数据集进行偏好对齐, 使用 DPO (Direct Preference Optimization) 优化
  • 文字渲染优化: 针对中英文文字渲染进行专项训练, 使用 OCR 数据增强和文字区域 mask 损失
  • 量化加速: 支持 FP8, INT8 量化, 降低推理成本。社区贡献了 GGUF,bnb 等量化方案

推理优化

  • 多 GPU 并行: FSDP + DeepSpeed Ulysses 序列并行, xDiT USP 加速
  • 显存优化: 模型卸载 (offload) 到 CPU, 梯度检查点, 潜在扩散 (latent diffusion)
  • 采样加速: 10-30 步 Flow Matching 采样, 支持 CFG (Classifier-Free Guidance)
  • 生态集成: ComfyUI, Diffusers, vLLM 等主流框架原生支持
社区生态:Wan2.1 开源后催生了大量社区工作 — Helios (分钟级视频生成), LightX2V (轻量加速), VACE (多模态控制), Wan-Move (精细运动控制, NeurIPS 2025), EchoShot (多镜头生成), AniCrafter (3DGS 角色动画), Phantom (多主体生成), UniAnimate-DiT (人体动画), CFG-Zero (CFG 优化), TeaCache (2× 加速), DiffSynth-Studio (量化+LoRA) 等 10+ 衍生项目。

🔬 损失函数（深入）

Flow Matching 损失 (主损失)

  • 训练目标: L_FM = E_{t,x0,ε} ||v_θ(x_t, t, c) - u_t(x0, x1)||²
  • 其中 v_θ 是模型预测的速度场 (velocity field), u_t 是目标速度 (从 x0 到 x1 的线性插值方向)
  • x_t = (1-t)·x0 + t·x1, t ∈ [0,1], x0 ~ N(0,I) 是高斯噪声, x1 是真实数据 (Wan-VAE 编码后的潜在表示)
  • c 是文本条件, 通过 umT5-XXL 编码后注入 DiT 交叉注意力层
  • Rectified Flow 优势: 相比标准 DDPM 的去噪目标, Rectified Flow 使用线性插值路径, 推理时可用更少步数 (10-30步) 生成高质量结果, 且训练更稳定
  • 与 DDPM 的区别: DDPM 预测噪声 ε, Flow Matching 预测速度 v = (x1 - x0); Rectified Flow 选择直线路径, 避免弯曲轨迹, 减少采样误差

Wan-VAE 损失 (预训练, 与主模型解耦)

  • 重建损失: L1 像素损失 + LPIPS 感知损失, 保证视觉质量
  • KL 散度: L_KL = D_KL(q(z|x) || N(0,I)), 约束潜在空间分布接近标准正态, 便于 Flow Matching 采样
  • 对抗损失: 判别器约束解码质量, 保证 1080P 视频的高保真重建。使用 PatchGAN 判别器
  • 时间一致性损失: 相邻帧的潜在表示之间的 L1 损失, 确保时序平滑

CFG (Classifier-Free Guidance)

  • 训练时: 随机丢弃文本条件 (10-15% 概率将 c 置为零向量), 学习无条件分布
  • 推理时: v_guided = v_uncond + s·(v_cond - v_uncond), s 为引导尺度 (推荐 6.0)
  • 采样参数: 1.3B 模型推荐 shift=8, guide_scale=6; 14B 模型推荐 shift=5, guide_scale=5
采样器设计:Wan2.1 使用 Flow Matching 采样器, 支持 adaptive step 和 fixed step 两种模式。推荐 30-50 步采样, 每步计算量与一次前向传播相当。Flow Matching 的直线路径使得大步长也能保持质量, 这是相比 DDPM 的核心优势。
Rectified Flow 选择原因:相比标准 DDPM, Rectified Flow 在推理时可用更少的步数(10-30步)生成高质量结果, 且训练更稳定, 这对于视频生成的高计算成本尤为重要。
