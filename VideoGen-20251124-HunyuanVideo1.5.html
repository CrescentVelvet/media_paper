📌 论文概要

HunyuanVideo 1.5 是腾讯混元团队推出的轻量级开源视频生成模型，仅用 8.3B 参数即可在消费级 GPU 上实现 SOTA 级别的视觉质量和运动连贯性。这是对初代 HunyuanVideo (13B+) 的重大升级——参数更少、质量更高、部署更友好。

核心成就：8.3B 参数达到开源视频生成模型新 SOTA，支持消费级 GPU 推理，统一 T2V + I2V 框架，多时长多分辨率生成。

SSTA 是核心架构创新：选择性注意力减少不相关 token 的计算，滑动窗口分块注意力 (Sliding Tile Attention) 在局部窗口内做密集注意力，兼顾全局视野和局部细节，显著降低长序列的计算复杂度。

渐进预训练与后训练

  • 渐进预训练：从低分辨率/短时长逐步扩展到高分辨率/长时长，降低训练成本
  • 后训练：在预训练基础上进行质量增强微调，提升视觉质量和运动连贯性
  • 统一框架：T2V 和 I2V 在同一模型中支持，多时长 (秒级) 多分辨率 (480P/720P)

🔬 技术细节

Step Distillation (步数蒸馏)

HunyuanVideo 1.5 提供步数蒸馏模型，将推理步数大幅缩减：

配置步数RTX 4090 端到端时间质量
原始模型~50 步~300 秒基线
Step Distill (推荐)8-12 步~75 秒与原始相当
Step Distill (极速)4 步~40 秒略有降低

端到端时间减少 75%，单张 RTX 4090 可在 75 秒内完成视频生成。

缓存推理

  • DeepCache：跨去噪步骤缓存中间特征，跳过冗余计算
  • TeaCache：时间感知缓存策略
  • TaylorCache：基于泰勒展开的缓存近似

训练基础设施

  • FSDP (Fully Sharded Data Parallel) 分布式训练
  • Context Parallel 支持长序列
  • 梯度检查点 (Gradient Checkpointing) 节省显存
  • Muon 优化器（区别于 AdamW）
  • 开放 LoRA 微调脚本

💡 个人分析

  • 轻量化路线：HunyuanVideo 1.5 走的是与 Kandinsky 5.0 Video Pro (19B) 相反的路线——更小参数、更高效率。8.3B 在消费级 GPU 上可运行，极大降低使用门槛
  • SSTA 架构创新：选择性 + 滑动窗口注意力是视频 DiT 的重要优化方向，类似思路在图像生成 (Sliding Tile Attention) 和长上下文 LLM 中也有应用。SSTA 让 8.3B 模型达到 13B+ 的质量
  • 工程成熟度高：Step distillation、3 种缓存策略、FP8、ComfyUI 集成、训练代码、LoRA 脚手架、Diffusers 支持——工程完备度在开源视频模型中首屈一指
  • Muon 优化器：使用 Muon 而非 AdamW，是视频生成领域少见的优化器选择。Muon 在某些场景下收敛更快、最终质量更好
  • Glyph-aware 编码：字形感知文本编码对中文/英文文字生成有重要意义，这是许多视频模型忽略的维度
  • 与 HunyuanVideo 1.0 的关系：1.5 不是简单升级，而是架构重新设计。参数减少 36% 但质量提升，说明 1.0 的架构效率有优化空间
  • 竞争定位：在 Wan2.1 (14B) 和 LTX-Video (2B) 之间，HunyuanVideo 1.5 (8.3B) 找到了中间定位——比 Wan2.1 更轻量，比 LTX 质量更高

📐 算法框图

HunyuanVideo 1.5 架构：基于 DiT 的视频生成模型，核心创新是 SSTA（Sparse-Scan Telescopic Attention）

Pipeline：
  • 文本编码：多语言 LLM 提取文本特征
  • 视频 VAE：3D VAE 编码视频到潜空间
  • SSTA-DiT：稀疏扫描望远镜注意力 + DiT 主干，兼顾全局和局部
  • Flow Matching：连续时间生成框架

SSTA 设计：通过稀疏扫描模式在空间和时间维度进行多尺度注意力，"望远镜"机制允许模型在粗粒度看全局、细粒度看局部，避免了全注意力的 O(n²) 计算开销。

💡 核心创新点

创新 1：反向缩参 — 从 1.0 的 13B 降到 8.3B（减少 36%），但质量不降反升，证明架构效率比堆参数更重要。
创新 2：SSTA 注意力 — Sparse-Scan Telescopic Attention，稀疏扫描+望远镜多尺度注意力，兼顾效率和质量。
创新 3：消费级 GPU 友好 — 8.3B 参数可在 RTX 4090 等消费级 GPU 上运行。
创新 4：Prompt 上采样 — 内置 prompt 增强机制，用户简短描述 → 模型自动扩写为详细描述。

🏋️ 训练策略

  • 从 1.0 迁移：从 HunyuanVideo 1.0 权重初始化，重新设计架构后继续训练
  • 渐进式训练：低分辨率 → 高分辨率，短视频 → 长视频
  • SSTA 预训练：在大规模数据上学习稀疏扫描模式
  • 后训练：美学优化、prompt 跟随能力提升
  • Prompt 上采样训练：训练模型自动扩写用户简短描述

📊 数据集构建

  • 大规模图像/视频文本对数据
  • 多分辨率、多长宽比训练数据
  • 美学评分模型筛选高质量数据
  • 运动质量评分过滤
  • Prompt 上采样训练数据：简短 prompt → 详细描述配对

📐 损失函数

  • Flow Matching 损失：L = E || v_θ(z_t, t, c) - (z1 - z0) ||²
  • VAE 损失：L1 + LPIPS + KL + 对抗损失
  • Prompt 上采样损失：交叉熵损失，训练模型扩写描述
关键设计：SSTA 的稀疏扫描不改变损失函数，而是改变了注意力的计算路径。通过减少不必要的长程注意力，在保持质量的前提下大幅降低计算量。
