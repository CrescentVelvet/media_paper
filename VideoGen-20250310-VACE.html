VACE: All-in-One Video Creation and Editing

📌 论文概要

VACE 是阿里巴巴 ali-vilab 推出的一体化视频创作与编辑框架，基于 Diffusion Transformer 范式。核心挑战在于视频生成需要同时保证时空一致性，VACE 通过统一接口将多种视频任务整合到一个模型中。

核心贡献：提出 Video Condition Unit (VCU) 统一接口和 Context Adapter 结构，使单一模型能够处理参考视频生成、视频编辑、遮罩视频编辑等多种任务，性能与专用模型相当，同时支持灵活的任务组合。

🏗️ 模型架构

Video Condition Unit (VCU)

VACE 的核心创新是 Video Condition Unit (VCU) —— 将不同任务的输入（编辑、参考、遮罩等）组织成统一的接口格式：

  • 将各种任务输入（编辑条件、参考帧、遮罩等）统一编码为结构化的时空表示
  • 不同任务通过 VCU 的不同配置来实现，无需修改模型结构
  • 支持任意视频合成任务的灵活组合

Context Adapter

通过 Context Adapter 结构将不同任务的概念注入模型：

  • 使用时间维和空间维的形式化表示来注入任务概念
  • Adapter 结构允许在不修改主模型的情况下引入新任务
  • 基于 Wan2.1 主干网络构建

支持的任务

任务输入输出
Reference-to-Video参考图像/视频 + 文本新视频
Video-to-Video Editing源视频 + 编辑指令编辑后视频
Masked Video-to-Video Editing源视频 + 遮罩 + 编辑指令局部编辑视频

🎯 关键特性

  • 统一框架：一个模型处理多种视频创作和编辑任务，无需为每个任务训练专用模型
  • 性能对标专用模型：统一模型在各子任务上达到与专用模型相当的性能
  • 灵活任务组合：支持多种任务的灵活组合，实现复杂创作流程
  • Wan2.1 生态：作为 Wan2.1 的配套工具，于 2025 年 5 月 14 日集成发布
  • 时空一致性：解决了视频编辑中内在的时空动态一致性挑战

⚙️ 技术细节

  • 基于 Wan2.1 的 DiT 主干和 Wan-VAE
  • VCU 统一了不同任务的输入组织方式
  • Context Adapter 通过时间和空间维度的形式化表示注入任务概念
  • 训练策略：多任务联合训练，共享主干参数
  • 推理时通过 VCU 配置切换任务类型，无需修改模型

📊 基准表现

  • 在多个视频编辑和生成子任务上，统一模型性能达到与专用模型相当水平
  • 支持多种应用的灵活组合，展示了统一框架的优势
  • 消融实验验证了 VCU 和 Context Adapter 各自的有效性

💡 个人分析

核心贡献

  • 统一接口设计：VCU 的设计思路优雅 —— 不同任务通过同一接口的不同配置实现，避免了多模型部署的复杂性
  • 实用价值：视频编辑是视频生成落地的重要场景，VACE 将生成与编辑统一在一个框架内，降低了实际应用的工程门槛
  • 与 Wan2.1 互补：VACE 是 Wan2.1 生态的重要组成部分，补齐了 Wan2.1 在编辑场景的能力

局限

  • 论文信息较少（GitHub README 获取失败），技术细节主要通过 arXiv 摘要和项目页面了解
  • 统一模型可能在极端场景下不如高度优化的专用模型
  • 任务组合的复杂度可能影响推理效率

VACE 架构：Video Condition Unit (VCU) + Context Adapter，统一视频创建与编辑

核心创新

  • VCU (Video Condition Unit)：将编辑、参考、遮罩等多种任务输入统一为一个接口
  • Context Adapter：通过时间和空间维度的形式化表示注入任务概念
  • 多任务统一：参考到视频生成、视频到视频编辑、遮罩视频编辑三合一

应用场景

场景具体应用
视频编辑视频风格转换、局部替换
参考生成从参考图像生成视频
遮罩编辑指定区域重新生成

训练策略与损失

  • 基于 Wan2.1 基座微调，多任务联合训练
  • Flow Matching 损失 + 任务一致性损失
  • 数据：视频编辑配对数据 + 参考生成数据
演进：VACE 代表了从“单一生成”到“多任务统一”的趋势。与 Cosmos 3 的多任务统一不同，VACE 聚焦于编辑场景，通过 VCU 统一接口实现灵活组合。
