VGGT: Visual Geometry Grounded Transformer

📄 arXiv: 2503.11651 📅 2025-03-14 🏢 Meta FAIR / Oxford VGG 🏆 CVPR 2025 Best Paper ✅ 开源

📋 论文概要

Feed-Forward 3D Transformer Camera Pose Depth Point Cloud Point Tracking Foundation Model

VGGT（Visual Geometry Grounded Transformer）是 Meta FAIR 联合 Oxford VGG 提出的前馈神经网络，可从一张、几张或数百张图像直接推断场景的所有关键 3D 属性，包括相机参数、点图（point maps）、深度图和 3D 点轨迹。该模型在 3D 计算机视觉领域迈出了重要一步——以往模型通常只针对单一任务设计，而 VGGT 将多个 3D 任务统一到一个模型中。

值得注意的是，VGGT 并非"Variable Quantity of Gaussians"的缩写，而是"Visual Geometry Grounded Transformer"——名称中的"VGG"致敬了 Oxford 的 Visual Geometry Group。该模型获得了 CVPR 2025 最佳论文奖。

核心贡献：单一前馈模型同时解决相机估计、多视角深度估计、密集点云重建和 3D 点追踪四大任务，在多项基准上超越需要后处理优化的传统方法，重建耗时不到 1 秒。

🔧 方法 / 架构

整体设计

VGGT 采用纯 Transformer 架构，接收任意数量的输入图像（1 到数百张），通过以下流程生成 3D 场景表征：

  • 图像编码：使用预训练的 DINOv2 作为视觉 backbone，将每张输入图像编码为 patch tokens
  • 多视角交互：所有图像的 tokens 在 Transformer 中进行全注意力交互（full self-attention），实现跨视角信息融合
  • 任务头：设计多个输出头来分别预测：相机参数（焦距、位姿）、点图、深度图、3D 点轨迹
  • 交替注意力：帧内自注意力与帧间交叉注意力交替进行，确保既捕获单帧细节又融合多视角信息

关键设计决策

  • 接地（Grounded）设计：将 Plücker 坐标等几何先验嵌入到 token 中，使网络对 3D 几何有显式感知
  • 可变输入数量：无需固定视角数量，从单张到数百张均可处理
  • 端到端训练：不需要 SfM 初始化，不需要任何测试时优化

🔬 技术细节

训练数据

在大规模 3D 数据集上训练，包括合成数据（如 CO3D、Objaverse 渲染）和真实世界数据（如 ScanNet、MegaDepth 等）。模型学习了通用的 3D 几何先验，而非场景特定优化。

Plücker 坐标接地

将相机的位姿信息以 Plücker 射线参数化嵌入到 token 中，使网络对几何有显式感知。后续研究发现（arXiv 2605.18599），这种将语义信息与空间信息混合的设计可能引入干扰，催生了语义-空间解耦的改进方案。

点图（Point Map）表示

不同于直接预测 3D 高斯参数（如 GS-LRM），VGGT 预测的是场景的点图——一种与视角对齐的 3D 坐标图。这种表示更通用，不绑定到特定的渲染管线。

衍生工作

  • QuantVGGT（arXiv 2509.21302, ICLR 2026）：4-bit 量化实现 3.7× 内存压缩 + 2.5× 加速，精度保持 98%
  • 4DLangVGGT（arXiv 2512.05060）：扩展到 4D 语言-视觉几何，用于动态场景理解
  • VGGT-SLAM 2.0：实时单目 SLAM 系统
  • MERG3R（arXiv 2603.02351）：分治框架解决大规模场景内存瓶颈

💡 个人分析

  • 3D 基础模型路线的里程碑：VGGT 证明了单一前馈模型可以同时解决多个 3D 任务并达到 SOTA，这是 3D 视觉从专用模型向通用基础模型演进的关键节点。CVPR 2025 最佳论文实至名归。
  • 与 DUSt3R/MASt3R 的传承：VGGT 继承了 DUSt3R（两视角 3D 重建）和 MASt3R（匹配+3D）的思想，将两视角扩展到任意数量视角，实现了真正的多视角统一。三者构成了一条清晰的技术演进线。
  • 与 GS-LRM 的路线对比：GS-LRM 直接预测 3D 高斯参数用于渲染，而 VGGT 预测更通用的点图表示。两者代表了前馈 3D 重建的两条路线——渲染导向 vs 几何导向。VGGT 的特征作为 backbone 用于下游任务的能力是其独特优势。
  • 规模化挑战：全注意力机制是 VGGT 的核心优势也是核心瓶颈——GPU 显存限制了可处理的图像数量。MERG3R 等分治框架是对这一限制的初步应对，未来可能需要稀疏注意力或分层架构。
  • 实际应用定位：对于需要快速获取 3D 几何信息的场景（AR/VR、机器人感知、场景理解），VGGT 提供了前所未有的速度和通用性。但对于需要极致精度的场景（精密测量、航测），传统 SfM+MVS 仍有价值。

🖼️ 算法框图与架构详解

Figure 1: 整体架构概览

VGGT 架构流水线:
  • Step 1 — Patchify: N 张输入图像 → DINOv2 backbone → patch tokens (16×16 patch, 每张图像 H/16 × W/16 个 token)
  • Step 2 — 相机 Token 拼接: 在每张图像的 patch token 序列前附加可学习的 camera token, 用于后续相机参数预测
  • Step 3 — 交替注意力 (Alternating Attention):
    • 帧内自注意力 (frame-wise): 每张图像的 token 独立做 self-attention, 捕获单帧内部细节, 计算量 O(N·L²)
    • 全局自注意力 (global): 所有图像的所有 token 一起做 self-attention, 融合多视角信息, 计算量 O((N·L)²)
    • 两种注意力交替堆叠, 帧内层捕获局部细节, 全局层融合跨视角信息
  • Step 4 — 多任务预测头:
    • Camera Head: 从 camera token 预测相机外参 (R, t) 和内参 (焦距 f)
    • DPT Head: 从视觉 token 预测深度图、点图、特征图 (用于点追踪)
  • Step 5 — 输出: 统一的 3D 场景表征 — 相机参数 + 深度图 + 点图 + 3D 点轨迹, 可直接用于渲染、追踪、SLAM

Figure 2: Plücker 坐标接地设计

几何先验注入:
  • 每个像素对应的相机射线以 Plücker 坐标参数化: (d, m) where d 是射线方向, m = o × d 是矩 (moment), o 是相机原点
  • Plücker 坐标被嵌入到 patch token 的位置嵌入中, 使网络对 3D 几何有显式感知
  • 区别于纯数据驱动的隐式学习 — 这是一种几何归纳偏置注入
  • 后续研究 (arXiv 2605.18599) 发现: 语义信息与空间信息混合可能引入干扰, 催生了语义-空间解耦的改进方案

Figure 3: 多任务预测头详解

预测头设计:
  • Camera Head: 线性层从 camera token 预测相机外参 (旋转 R 使用 6D 表示, 平移 t 为 3D 向量) 和内参 (焦距 f)
  • Dense Prediction Head (DPT): 基于 DPT (Dense Prediction Transformer) 架构, 从多尺度 token 重建稠密输出:
    • 深度图: 每像素的深度值 d ∈ ℝ (尺度不变)
    • 点图: 每像素的 3D 坐标 (X, Y, Z) ∈ ℝ³ (世界坐标系)
    • 特征图: 用于下游点追踪任务的高维特征
  • 点追踪: 从不同视角的特征图进行特征匹配, 输出 3D 点轨迹 + 可见性标签

过完整预测 (Over-complete Predictions) 策略

关键设计 — 冗余预测提升精度:VGGT 同时预测深度图和点图, 虽然点图可以从深度+相机参数推导, 但实验表明独立预测点图再从深度+相机推导点图有更好的精度。这种“过完整”预测策略让模型从多个角度学习 3D 几何, 互相补充。

💡 核心创新点

创新 1: 多任务统一前馈模型
首次在单一前馈模型中同时解决相机估计、多视角深度、密集点云重建和 3D 点追踪四大任务, 以前这些任务需要各自的专用模型和后处理优化。
创新 2: Plücker 坐标接地
将相机位姿信息以 Plücker 射线参数化嵌入到 token 中, 使网络对 3D 几何有显式感知。这是一种几何先验注入, 区别于纯数据驱动的隐式学习。
创新 3: 可变视角数量
无需固定视角数量, 从 1 张到数百张图像均可输入。全注意力机制天然支持任意序列长度, 通过交替帧内/帧间注意力高效处理。
创新 4: 亚秒级速度
重建数百张图像不到 1 秒, 比传统 SfM+MVS (COLMAP, 需要数小时) 快 4-5 个数量级, 使实时 3D 感知成为可能。
创新 5: backbone 特征可迁移
VGGT 的预训练特征可增强下游任务(非刚性追踪、前馈新视角合成), 作为 3D 基础模型的特征提取器使用, 类似 2D 领域的 DINOv2。

🏋️ 训练策略（深入）

阶段 1: 主干预训练 — 多任务联合学习

  • 视觉 backbone: 使用预训练 DINOv2 (ViT-Large) 作为图像编码器, 继承 2D 视觉理解能力。DINOv2 参数在训练中冻结或微调
  • 多任务联合训练: 相机估计 + 深度 + 点云 + 点追踪同时训练, 共享 backbone。实验表明多任务联合训练比单任务训练有更好的精度, 因为相关 3D 属性互相补充
  • 几何接地: Plücker 坐标在训练中嵌入到 token 位置嵌入, 使模型学习几何先验而非纯模式匹配
  • 数据多样性: 合成数据(CO3D, Objaverse 渲染) + 真实数据(ScanNet, MegaDepth)混合训练, 覆盖室内/室外/物体级/场景级多种场景
  • 训练规模: 大规模 3D 数据集训练, 学习通用 3D 几何先验而非场景特定优化

训练细节

  • 交替注意力训练: 帧内和帧间注意力层交替堆叠 (如 [frame-wise, global, frame-wise, global, ...]), 逐层加深多视角融合
  • 任务权重: 多个任务头的损失按一定权重组合, 自动平衡各任务。相机损失的权重较小, 深度和点图损失的权重较大
  • 坐标归一化: 对 GT 坐标进行归一化处理, 使模型预测相对坐标而非绝对坐标, 提高泛化能力
  • 过完整预测策略: 同时预测深度图和点图 (虽然点图可以从深度+相机推导), 让模型从多个角度学习 3D 几何, 互相补充

阶段 2: 下游任务微调

  • 新视角合成: 用 VGGT 特征作为 backbone, 微调生成新视角的网络
  • 动态点追踪: 用 VGGT 特征增强现有点追踪器 (如 CoTracker), 在动态场景中追踪点
  • BA 后处理 (可选): 将 VGGT 预测作为初始值, 用可微 Bundle Adjustment 进一步优化, 达到最高精度
消融实验关键发现:
  • 多任务学习正向迁移: 同时学习相机+深度+点图+追踪比单任务训练精度更高
  • 交替注意力优于纯全局注意力: 帧内层捕获局部细节, 全局层融合跨视角信息, 两者互补
  • Plücker 接地有效但可改进: 后续研究 (arXiv 2605.18599) 发现语义-空间解耦更优
  • 从深度+相机推导点图比直接预测点图更精确: 启示过完整预测的价值

🔬 损失函数（深入）

相机参数损失

  • 旋转损失: L_R = ||R_pred - R_gt||_F², 使用 Frobenius 范数。R 使用 6D 表示 (Zhou et al. 2019), 比 4D 四元数更稳定
  • 平移损失: L_t = ||t_pred - t_gt||², 欧式距离
  • 焦距损失: L_focal = |f_pred - f_gt| / f_gt, 相对误差避免绝对尺度偏差
  • 位姿总损失: L_pose = L_R + L_t + L_focal

深度损失 (尺度不变)

  • Scale-invariant L1: L_depth = ||d_pred / s - d_gt||_1, s = median(d_pred / d_gt) 是尺度因子
  • 设计原因: 尺度不变设计使模型不依赖绝对尺度标注, 允许从不同尺度的数据集混合训练 (合成数据有真实尺度, 真实数据可能只有相对深度)
  • 额外约束: 深度梯度的 L1 损失, 保证深度边缘的清晰度

点图损失

  • 3D 点 L1: L_point = ||P_pred - P_gt||_1, 逐像素 3D 坐标回归
  • 多视角一致性: 不同视角预测的点图应一致 — 将所有视角的点图变换到同一坐标系后计算 L1 损失
  • 过完整策略: 同时预测独立点图和从深度+相机推导的点图, 取两者加权平均作为最终输出

点追踪损失

  • 轨迹 L2: L_track = ||T_pred - T_gt||², 3D 点轨迹回归
  • 可见性分类: 二元交叉熵 L_vis = -[y·log(p) + (1-y)·log(1-p)], 判断点是否可见 (被遮挡时 y=0)

总损失

  • L_total = λ_pose·L_pose + λ_depth·L_depth + λ_point·L_point + λ_track·(L_track + L_vis)
  • 权重设置: λ_pose 较小 (相机预测相对容易), λ_depth 和 λ_point 较大 (稠密预测更难)
  • 训练策略: 所有任务同时训练, 不需要分阶段训练, 简化训练流程
Plücker 接地的作用与局限:Plücker 坐标将相机射线参数化注入 token, 提供几何先验。但后续研究 (arXiv 2605.18599) 发现语义信息与空间信息混合可能引入干扰 — 当场景纹理丰富时, 几何信息可能被纹理信息覆盖。语义-空间解耦的改进方案将两者分离到不同维度, 获得更好的性能。这表明几何接地虽然有效但仍有优化空间。

VGGT 衍生生态

  • QuantVGGT (arXiv 2509.21302, ICLR 2026): 4-bit 量化, 3.7× 内存压缩 + 2.5× 加速, 精度保持 98%。使 VGGT 可在消费级 GPU 上运行
  • 4DLangVGGT (arXiv 2512.05060): 扩展到 4D 语言-视觉几何, 用于动态场景理解。将语言特征注入 VGGT backbone, 实现 4D 场景的语言定位
  • VGGT-SLAM 2.0: 实时单目 SLAM 系统, 利用 VGGT 的前馈推理替代传统特征匹配+BA 流程
  • MERG3R (arXiv 2603.02351): 分治框架, 将大规模场景分割为子集分别处理再合并, 解决全注意力的 O(N²) 显存瓶颈

📐 算法框图

VGGT 架构概览：给定任意数量的输入图像，通过单次前向传播生成 3D 几何表示。

Pipeline：
  • 输入：N 张图像 + 相机参数（可选）
  • 图像编码：DINOv2 提取每张图像的 patch 特征
  • Plücker 坐标接地：将相机射线方向编码为 Plücker 参数，注入到 token 中
  • 交替注意力：帧内 self-attention + 帧间 cross-attention 交替进行
  • 输出：点图（Point Map）+ 深度图 + 相机位姿 + 3D 高斯参数（可选）

核心设计图：多视角图像 → DINOv2 编码 → Plücker 接地 → 交替 Transformer → 多头输出（点图/深度/位姿/高斯）

关键创新：用点图（Point Map）替代直接预测高斯参数，不绑定渲染管线，更通用。

💡 核心创新点

创新 1：单前向 3D 重建 — 无需 per-scene 优化，单次前向传播即可从任意数量图像重建 3D 几何，颠覆传统 SfM+MVS pipeline。
创新 2：可变数量输入 — 不限制输入图像数量，1 张到 100+ 张均可处理，通过交替注意力机制实现可扩展性。
创新 3：Plücker 坐标接地 — 将相机位姿以射线参数化嵌入 token，让网络显式感知几何。后续研究发现语义-空间混合可能干扰，催生了解耦改进。
创新 4：点图表示 — 预测与视角对齐的 3D 坐标图，比直接预测高斯参数更通用，不绑定渲染管线。
创新 5：多头输出统一 — 点图 + 深度 + 位姿 + 高斯参数，一个模型同时输出多种 3D 表示。

🏋️ 训练策略

训练阶段

  • 骨干初始化：从 DINOv2 预训练权重初始化图像编码器
  • 多数据集联合训练：合成数据（CO3D、Objaverse 渲染）+ 真实数据（ScanNet、MegaDepth、BlendedMVS 等）
  • 课程学习：从 2 视角逐步增加到多视角
  • 数据增强：随机裁剪、颜色抖动、相机扰动

关键设计

  • 交替注意力：帧内 self-attention 提取单帧特征，帧间 cross-attention 融合多视角信息，交替堆叠
  • 可变视角数：不固定输入数量，通过 attention mask 实现任意数量输入
  • 端到端训练：从图像到 3D 输出全流程可微，无中间手工模块

📐 损失函数

点图损失

  • L1 回归损失：L_point = || M_pred - M_gt ||₁，M 为点图坐标
  • 尺度不变损失：对预测和 GT 点图做归一化后计算，消除尺度歧义

深度损失

  • SILog（Scale-Invariant Log Loss）：L_depth = || d - d̂ ||² - λ·(|| d - d̂ ||₁)²
  • 在 log 空间计算，对尺度不敏感

相机位姿损失

  • 位姿回归损失：L_pose = || R_pred - R_gt ||² + || t_pred - t_gt ||²
  • 旋转用四元数表示避免欧拉角奇异性

高斯参数损失（可选）

  • 高斯属性回归：位置 + 不透明度 + 协方差 + 球谐系数，各子项加权 L2
总损失：L = λ₁·L_point + λ₂·L_depth + λ₃·L_pose + λ₄·L_gaussian，各权重通过实验调优。

演进脉络

  • SfM+MVS（2010s）：手工特征匹配 + 迭代优化，慢但精确
  • LRM（2023）：单图前馈重建，但需要 per-object 优化
  • GS-LRM / Long-LRM（2024）：多视角前馈 → 直接预测高斯参数
  • VGGT（2025）：任意数量图像 → 多头输出，点图表示更通用
  • QuantVGGT（2026）：量化部署优化，ICLR 2026
  • 语义-空间解耦（2026）：解决 Plücker 接地的语义干扰问题
趋势：从“迭代优化”→“单前向预测”→“任意数量输入+多头输出”→“量化部署+解耦改进”。VGGT 是前馈重建的里程碑，标志着 3D 重建从优化范式转向预测范式。
